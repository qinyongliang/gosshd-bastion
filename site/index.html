How it works

SSH that understands intent, not just identity

Replace raw key access with a control plane. Agents only know the target service alias; GOSSHD resolves it to the actual SSH service in a personal or organization scope.

01

Register private nodes

Connect on-prem, VPC, and edge machines without exposing ports. Once a private node registers, it is just another SSH service and can be renamed to a business alias.

02

Bind labels & groups

Map users, service accounts, and agents to labels. Grant least-privilege access by environment, role, or task.

03

Review every command

Rules and LLM review run before forwarding. Allowed commands continue to the target; high-risk commands are rejected and return SSH exit code 126.

04

Replay & audit

Recorded sessions include timestamps, keystrokes, and decisions. Ship evidence to compliance and incident review.

AI Command Review

One target alias. Different commands. Different outcomes.

The caller uses standard SSH exec. Safe commands are forwarded to the target host; high-risk commands never enter the target shell and return a denial plus exit code.

aws-ap-sg-billing-db@gosshd.site
Allowed 09:14:02.104
aws-ap-sg-billing-db

The public key identifies the operator, then the alias resolves to the billing database service. REINDEX is treated as maintenance and audited with exit_code=0.

Blocked 09:14:08.831
DROP TABLE customers;

Destructive schema change detected. The command is not forwarded to the target host; SSH exec returns exit_code=126.

Audited 09:14:08.833
Audit #A-2938

The audit database stores operator, public key, target alias, resolved target, command, decision, reason, and exit code for human review.

Capabilities

Built for machines that need to touch infrastructure

GOSSHD Bastion turns agent-to-server access into a governed, observable workflow. No open ports. No shared keys. No blind trust.

Private node registration

Nodes connect outbound to the bastion. Keep servers dark to the public internet while agents reach them on demand.

SSH access control

Enforce who and what can reach each node. Tie permissions to labels, groups, and service identity instead of scattered authorized_keys.

Label & user-group policy

Compose rules from simple labels. Allow the backup agent to touch storage nodes, the deploy agent to touch compute, and nothing else.

LLM real-time command review

The model reads intent before a command is forwarded. Combine it with deterministic rules: allowed commands run, denied commands return 126.

Terminal recording & replay

Capture target alias, resolved target, command output, and exit code on a precise timeline. Rebuild incidents without grepping through text.

Human audit evidence

Export decision logs, replay files, and policy traces. Make security reviews a conversation with proof, not a scavenger hunt.
