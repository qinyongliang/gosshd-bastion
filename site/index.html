Control plane

One gateway for human operators and autonomous AI work.

GOSSHD Bastion keeps the SSH interface familiar while moving identity, target routing, command policy, and audit into a SQLite-backed control plane.

01

Public-key identity

SSH keys identify platform users. The SSH username becomes the target alias, so access stays simple.

02

Agent-backed services

Private hosts enroll through install scripts and become normal renameable SSH targets.

03

Command security groups

Bind whitelist, blacklist, or LLM-backed review policies to targets and organization user groups.

04

Audit for automation

Every exec attempt records user, target, decision, reason, timing, exit code, and remote address.

Alias route

ssh inference-gpu@public-ip

The presented key finds the user. The alias finds a personal or organization target. A policy decides whether the command runs locally, through an agent, or gets denied before it reaches production.

AI safety posture

Designed for tools that can act faster than humans can review.

Default group model

Every organization has an all-members group, and policies can target one or more groups.

Fail-closed LLM review

OpenAI-compatible review can decide unmatched commands. Invalid responses are denied.

Startup agents

Linux agents install with systemd. Windows agents install with sc.exe for boot-time availability.

First launch

Bring the bastion online.

gosshd-server \
  --http-listen :18080 \
  --ssh-listen :22022 \
  --database-path ./data/gosshd.db \
  --host-key-path ./data/gosshd_host_key \
  --public-host bastion.example.com:18080
