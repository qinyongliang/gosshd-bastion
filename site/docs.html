Field guide

Deploy the SSH control plane built for AI agents, MCP, and automation.

GOSSHD Bastion ships as one Go server with SQLite databases, an embedded web console, an SSH gateway, private-node enrollment, command safety groups, LLM review, and terminal replay metadata for a command-level SSH safety gateway.

Deploy the server See the review flow
01

Deploy the SSH control plane

Download the server package from GitHub Releases, give it persistent paths, and set the public host used by private-node install commands. 

gosshd-server \
  --http-listen :18080 \
  --ssh-listen :22022 \
  --database-path ./data/gosshd.db \
  --audit-database-path ./data/gosshd-audit.db \
  --host-key-path ./data/gosshd_host_key \
  --agent-cache-path ./agent-cache \
  --public-host bastion.example.com:18080 \
  --bootstrap-admin-password 'change-me'

Open http://bastion.example.com:18080/, sign in as admin, add your public key, and create an organization or use your personal organization.

02

Connect by SSH alias

Direct SSH server

Add a host, port, remote username, auth method, display name, alias, and colored tags.

Private node

Create an enrollment token, install the node, then rename and tag it like any other SSH service.

ssh -p 22022 inference-gpu@bastion.example.com hostname

The presented SSH public key maps to a platform user. The SSH username maps to a target alias in the current personal or organization scope.

03

Install private nodes

The install guide generated by the console includes tokenized commands for each platform.

curl -fsSL http://bastion.example.com:18080/install/<token>.sh | sh
irm http://bastion.example.com:18080/install/<token>.ps1 | iex

Use startup-service mode for long-lived nodes.

curl -fsSL http://bastion.example.com:18080/install/<token>.sh | sudo sh -s -- install
$s='http://bastion.example.com:18080/install/<token>.ps1'
irm $s -OutFile $env:TEMP\gosshd-agent-install.ps1
powershell -ExecutionPolicy Bypass -File $env:TEMP\gosshd-agent-install.ps1 -Install

Linux installs through systemd. Windows installs through sc.exe.

04

Review commands before execution

Command safety groups can bind to one or many targets, target tags, organization user groups, and source IP ranges. They can also enable or block interactive terminal, upload, download, and port forwarding.

  1. Blacklist rules deny exact, prefix, or contains matches.
  2. Whitelist rules allow exact, prefix, or contains matches.
  3. Unmatched commands can be sent to an OpenAI-compatible model.
  4. Allowed LLM responses can omit a reason; denied responses should explain why.
05

Replay audit sessions

Exec, shell, SFTP, and port-forward decisions are written to the audit database. Interactive terminal recordings are stored as compressed replay files with timestamps, so the web console can reconstruct what happened instead of showing only a command line.

00:00:00.000 ssh inference-gpu@bastion 00:00:07.284 docker ps --format '{{.Names}} {{.Status}}' 00:00:08.704 llm allow (1.4s)
06

Automate through MCP

The /mcp endpoint exposes registration, organizations, public keys, SSH services, target tags, enrollments, LLM configs, prompt resources, command policies, policy bindings, and audit queries to tools. 

http://bastion.example.com:18080/mcp
