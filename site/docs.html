Documentation

Operate SSH access for AI services without giving automation a blind shell.

GOSSHD Bastion is a single Go server with SQLite storage, an embedded console, and an SSH endpoint. It turns users, organizations, public keys, direct targets, agent targets, command safety groups, LLM review, and audit logs into one deployable control plane.

Quickstart

  1. Download a server archive from the release page.
  2. Start the server with HTTP, SSH, database, and host-key paths.
  3. Open the console, register a user, and add your SSH public key.
  4. Add a direct target or enroll an agent target.
  5. Connect with the target alias as the SSH username.
gosshd-server \
  --http-listen :80 \
  --ssh-listen :22 \
  --database-path gosshd.db \
  --host-key-path gosshd_host_key
ssh test2@public-host

Identity model

Users authenticate to the web console with email and password. Users authenticate to SSH with their configured public keys. During SSH login, the presented public key maps to a platform user, and the SSH username maps to a target alias.

Every registered user receives a personal organization. Shared organizations can invite users, and every organization includes a default all-members group for policy binding.

SSH targets

Direct target

The bastion connects directly to a host and port using password or private-key authentication.

Agent target

The bastion opens a TCP stream through an enrolled agent, then performs the SSH handshake over it.

Agent-enrolled hosts are normal SSH targets. They can be renamed, bound to policies, and filtered in audit views just like manually added targets.

Agent install

Create an enrollment in the console. The default command runs the agent in the current shell.

curl -fsSL http://public-host/install/<token>.sh | sh
irm http://public-host/install/<token>.ps1 | iex

Use install mode when the agent should survive restarts.

curl -fsSL http://public-host/install/<token>.sh | sudo sh -s -- install
$s='http://public-host/install/<token>.ps1'; irm $s -OutFile $env:TEMP\gosshd-agent-install.ps1; powershell -ExecutionPolicy Bypass -File $env:TEMP\gosshd-agent-install.ps1 -Install

Linux uses systemd through systemctl. Windows uses sc.exe.

Command safety groups

Policies bind to targets and optionally to organization user groups. A policy with no user-group binding applies to all users who can access the target.

  1. Whitelist rules allow exact, prefix, or contains matches.
  2. Blacklist rules deny exact, prefix, or contains matches.
  3. Unmatched commands can be sent to LLM review.
  4. If no LLM is configured, the policy default action is used.

LLM review

LLM configs use an OpenAI-compatible chat completions contract with base URL, API key, model, timeout, and prompt resource. The model response must be JSON with allow and reason. Invalid JSON and request errors fail closed.

Audit logs

Exec attempts record user, target, organization, session, command, request type, decision, reason, exit code, timing, and remote address. The audit table is built for AI operations where command provenance matters after the fact.

Release packages

The current release publishes cross-platform server archives and standalone agent binaries. This version intentionally does not publish full packages.
