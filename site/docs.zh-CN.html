文档

让 AI 服务使用 SSH，但不给自动化一条无审计的盲 shell。

GOSSHD Bastion 是一个带 SQLite 存储、内置控制台和 SSH 入口的 Go 服务。 它把用户、组织、公钥、直连目标、Agent 目标、命令安全组、LLM 审核和审计日志放入同一个可部署控制平面。

快速开始

  1. 从发布页下载 server 压缩包。
  2. 使用 HTTP、SSH、数据库和 host key 路径启动服务。
  3. 打开控制台，注册用户，并添加自己的 SSH 公钥。
  4. 添加直连目标，或注册 Agent 目标。
  5. 把目标别名作为 SSH 用户名执行 SSH 命令。
gosshd-server \
  --http-listen :18080 \
  --ssh-listen :22022 \
  --database-path ./data/gosshd.db \
  --host-key-path ./data/gosshd_host_key \
  --public-host bastion.example.com:18080
ssh -p 22022 test2@bastion.example.com hostname

身份模型

用户通过邮箱和密码登录 Web 控制台，通过自己配置的 SSH 公钥登录 SSH 入口。SSH 登录时， 提交的公钥映射到平台用户，SSH 用户名映射到目标别名。

每个注册用户都会获得个人组织。共享组织可以邀请用户，并且每个组织都包含默认全员用户组，用于绑定策略。

SSH 目标

直连目标

堡垒机使用账号密码或私钥认证，直接连接到指定主机和端口。

Agent 目标

堡垒机通过已注册的 Agent 打开 TCP 流，然后在这条流上完成 SSH 握手。

通过 Agent 注册上来的主机也是普通 SSH 目标。它们可以重命名、绑定策略，也可以像手动添加的目标一样在审计视图里过滤。

Agent 安装

在控制台的新建 SSH 目标页面进入 SSH Agent 标签页，创建注册令牌并复制引导命令。默认命令会在当前 shell 中运行 Agent。

curl -fsSL http://bastion.example.com:18080/install/<token>.sh | sh
irm http://bastion.example.com:18080/install/<token>.ps1 | iex

需要开机启动时使用 install 模式。

curl -fsSL http://bastion.example.com:18080/install/<token>.sh | sudo sh -s -- install
$s='http://bastion.example.com:18080/install/<token>.ps1'; irm $s -OutFile $env:TEMP\gosshd-agent-install.ps1; powershell -ExecutionPolicy Bypass -File $env:TEMP\gosshd-agent-install.ps1 -Install

Linux 安装脚本通过 systemd 和 systemctl 配置服务。Windows 安装脚本通过 sc.exe 配置服务。

命令安全组

策略可以绑定到单个或多个 SSH 目标，也可以绑定到一个或多个组织用户组。没有用户组绑定的策略会应用到所有可访问目标的用户。

  1. 白名单规则允许 exact、prefix 或 contains 匹配。
  2. 黑名单规则拒绝 exact、prefix 或 contains 匹配。
  3. 未命中的命令可以发送给 LLM 实时审核。
  4. 如果没有配置 LLM，则使用策略默认动作。

LLM 审核

LLM 配置使用 OpenAI 兼容的 chat completions 协议，包含 base URL、API key、模型、超时和提示词资源。 模型响应必须是带 allow 和 reason 字段的 JSON。无效 JSON 和请求错误都会默认拒绝。

审计日志

每次执行请求都会记录用户、目标、组织、会话、命令、请求类型、决策、原因、退出码、耗时和远端地址。 审计表面向 AI 运维场景设计，方便在事后追踪命令来源和策略判断。

发布包

当前版本发布跨平台 server 压缩包和独立 agent 二进制。本版本暂不发布 full 包。
