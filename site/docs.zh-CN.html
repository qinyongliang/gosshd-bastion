落地指南

在 AI 服务前放置一个可审核的 SSH 控制平面。

GOSSHD Bastion 以单个 Go 服务交付，包含 SQLite 数据库、内置 Web 控制台、SSH 网关、 私有节点注册、命令安全组、LLM 审核和终端回放元数据。

部署服务 查看审核链路
01

部署堡垒机

从 GitHub Releases 下载 server 包，配置持久化路径，并设置私有节点安装命令会使用的 public host。 

gosshd-server \
  --http-listen :18080 \
  --ssh-listen :22022 \
  --database-path ./data/gosshd.db \
  --audit-database-path ./data/gosshd-audit.db \
  --host-key-path ./data/gosshd_host_key \
  --agent-cache-path ./agent-cache \
  --public-host bastion.example.com:18080 \
  --bootstrap-admin-password 'change-me'

打开 http://bastion.example.com:18080/，用 admin 登录，添加自己的公钥， 然后创建组织或使用个人组织。

02

通过 SSH 别名连接

直连 SSH 服务器

添加主机、端口、远程用户名、认证方式、展示名称、别名和彩色标签。

私有节点

创建注册令牌，完成安装后，它也可以像普通 SSH 服务一样重命名和打标签。

ssh -p 22022 inference-gpu@bastion.example.com hostname

SSH 公钥会映射到平台用户。SSH 用户名会在当前个人或组织范围中解析为目标别名。

03

安装私有节点

控制台生成的安装引导会包含当前令牌和平台对应命令。

curl -fsSL http://bastion.example.com:18080/install/<token>.sh | sh
irm http://bastion.example.com:18080/install/<token>.ps1 | iex

长期运行的节点使用开机启动模式。

curl -fsSL http://bastion.example.com:18080/install/<token>.sh | sudo sh -s -- install
$s='http://bastion.example.com:18080/install/<token>.ps1'
irm $s -OutFile $env:TEMP\gosshd-agent-install.ps1
powershell -ExecutionPolicy Bypass -File $env:TEMP\gosshd-agent-install.ps1 -Install

Linux 通过 systemd 安装。Windows 通过 sc.exe 安装。

04

执行前审核命令

命令安全组可以绑定到一个或多个目标、目标标签、组织用户组和来源 IP 范围。它们也能控制交互式终端、 上传、下载和端口转发能力。

  1. 黑名单规则拒绝 exact、prefix 或 contains 匹配。
  2. 白名单规则允许 exact、prefix 或 contains 匹配。
  3. 未命中的命令可以交给 OpenAI 兼容模型审核。
  4. LLM 通过时可以省略原因；拒绝时应返回原因。
05

回放审计会话

Exec、Shell、SFTP 和端口转发决策会写入审计数据库。交互式终端录制会以压缩回放文件存储并带时间戳， 所以 Web 控制台能还原发生过程，而不只是展示一条命令。

00:00:00.000 ssh inference-gpu@bastion 00:00:07.284 docker ps --format '{{.Names}} {{.Status}}' 00:00:08.704 llm allow (1.4s)
06

通过 MCP 自动化

/mcp 端点暴露注册、组织、公钥、SSH 服务、目标标签、私有节点注册、LLM 配置、 提示词资源、命令安全组、策略绑定和审计查询。 

http://bastion.example.com:18080/mcp
