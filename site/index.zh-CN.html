为 AI 服务的堡垒机

GOSSHD Bastion

让 AI 智能体和运维人员以原生 SSH 的方式访问私有服务，同时经过别名路由、命令策略、LLM 审核和可回放审计。

几分钟启动 查看访问链路
SSH 入口在线 LLM 守卫已启用 终端录制运行中
01 公钥识别当前用户
02 SSH 用户名解析目标别名
03 命令策略和 LLM 给出决策
04 执行和交互终端进入审计

远程访问，可被回放

使用 SSH 的旧习惯，获得面向 AI 的新审计。

运维人员和 AI 任务继续使用熟悉的命令。GOSSHD Bastion 负责把公钥映射到用户，把别名解析到目标， 执行安全组决策，并保存带时间戳的终端录制数据。

LLM 命令审核

每条命令抵达生产环境前，都要先得到策略结论。

黑白名单先做确定性判断。未命中的命令可以交给 OpenAI 兼容模型，使用你的提示词、超时时间和默认拒绝策略。

身份 public key -> mia@ops
范围 production-ai / readonly group
命令 docker ps | head
决策 allow (1.4s)

内置控制台

控制平面随服务一起启动。

在内置 Web UI 中管理组织、用户组、公钥、SSH 服务、彩色标签、私有节点、策略绑定、LLM 配置、 审计搜索和终端回放。

SSH

别名原生访问

用户通过 ssh alias@public-ip 连接。别名可以指向直连主机，也可以指向私有节点。

NODE

私有服务注册

带令牌的安装命令可注册 NAT 后方主机，并在 Linux 与 Windows 上安装开机启动服务。

SAFE

命令安全组

按目标、标签、用户组、来源 IP 范围和终端/上传等能力开关绑定规则。

MCP

自动化控制面

通过 MCP 暴露注册、目标、标签、策略、提示词、LLM 配置和审计查询。

首次启动

启动堡垒机。

gosshd-server \
  --http-listen :18080 \
  --ssh-listen :22022 \
  --database-path ./data/gosshd.db \
  --host-key-path ./data/gosshd_host_key \
  --public-host bastion.example.com:18080
打开快速开始
