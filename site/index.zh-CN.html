工作原理

理解意图的 SSH，不只是验证身份

用控制平面取代原始密钥访问。AI 只知道要访问的目标服务别名，GOSSHD 再把它解析到用户或组织下的真实 SSH 服务。

01

注册私有节点

连接本地机房、VPC 与边缘设备，无需暴露端口。私有节点注册后会成为普通 SSH 服务，也可以重命名成业务别名。

02

绑定标签与用户组

将用户、服务账号和 Agent 映射到标签。按环境、角色或任务授予最小权限访问。

03

逐条审查命令

规则和 LLM 在命令转发前完成判定。允许的命令继续执行；高危命令直接拒绝，并向 SSH 客户端返回退出码 126。

04

回放与审计

录制的会话包含时间戳、按键与决策结果。为合规与事件复盘提供完整证据。

AI 命令审查

同一个业务别名，不同命令，不同结果

核心能力

为需要触碰基础设施的 AI 而设计

GOSSHD Bastion 将 Agent 到服务器的访问转化为可治理、可观测的工作流。没有开放端口，没有共享密钥，没有盲目信任。

私有节点注册

节点主动向安全网关发起出站连接。服务器可对公网隐身，而 Agent 能按需访问。

SSH 访问控制

精确控制谁、什么可以访问哪个节点。用标签、用户组与服务身份替代零散的 authorized_keys。

标签与用户组策略

用简单标签组合策略。允许备份 Agent 访问存储节点，部署 Agent 访问计算节点，其他一概不行。

LLM 实时命令审查

模型在命令转发前读懂意图并标记风险。与确定性规则结合，允许就执行，拒绝就返回 126。

终端录制与回放

以精确时间轴记录目标别名、真实目标、命令输出和退出码。复盘事件、训练模型、应对审计都能回到现场。

人工审计证据

导出决策日志、回放文件与策略追踪。让安全审查成为有凭有据的对话，而不是寻宝游戏。
